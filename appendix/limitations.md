# Limitations And Upgrade Path

## Limitations

This case is limited by public information. It should be read as a portfolio case study, not as a final board recommendation or investment opinion.

Key limitations:

- No internal unit economics for Taiwan, Eats/WOW, or Farfetch.
- No stakeholder interviews.
- No internal management budget or capital allocation constraints.
- No full 2026-2028 cash-flow scenario model.
- No designed slide deck or data visualization layer.
- No legal analysis of regulatory matters.

## Data Needed For A Stronger Case

| Area | Data needed |
| --- | --- |
| Product Commerce | Post-incident recovery trend, voucher redemption curve, capacity utilization, gross margin bridge. |
| Taiwan | Revenue, gross margin, EBITDA, cohort retention, order frequency, fulfillment density, loss intensity by city/cohort. |
| Eats/WOW | Contribution margin, subsidy intensity, delivery cost, member retention, churn, incremental spend, regulatory scenario impact. |
| Farfetch | Standalone revenue, gross margin, adjusted EBITDA, restructuring savings, integration milestones, control remediation status. |
| Capital allocation | Loss budget, capex needs, buyback hurdle, management bandwidth constraints. |

## Upgrade Path

1. Build a 2026-2028 base/upside/downside scenario model.
2. Replace public-data scorecard assumptions with internal initiative economics.
3. Convert the deck outline into 8-10 designed slides.
4. Add backup slides for scorecard methodology, source ledger, and sensitivity.
5. Add a 90-day implementation plan with RACI, dashboard definitions, and escalation thresholds.

## Quality Read

This is a strong public-data practice case. It is not equivalent to a live consulting project. The largest gap is data access, not structure.
